# Create a wallet for your agent
curl -X POST https://api.agenttool.dev/economy/v1/wallets \
  -H "Authorization: Bearer at_..." \
  -d '{"name": "agent-primary", "agentId": "gpt-4o-agent-1"}'

# Set a spending policy — agent can't bypass this
curl -X PUT .../wallets/<id>/policy \
  -d '{"maxPerTransaction": 100, "maxPerDay": 2000}'

# Agent spends autonomously
curl -X POST .../wallets/<id>/spend \
  -d '{"amount": 50, "counterparty": "agent-search", "description": "web search"}'
Features

Everything an agent economy needs

💳

Per-agent wallets

Each agent gets its own balance. Fund via Stripe checkout or USDC on Base. Track every spend with a full transaction log.

🔐

Spending policies

Set per-transaction, hourly, and daily limits. Whitelist recipients. Require approval above a threshold. All enforced atomically.

🤝

Agent-to-agent escrow

Agent A hires Agent B. Funds lock. B does the work. A releases — or disputes. No trust required. The protocol enforces payment.

⚡

Atomic transactions

Spend is SELECT FOR UPDATE inside a DB transaction. No race conditions. No double-spends. Safe at any concurrency.

📊

Rate-limited aggregates

Hourly and daily spend totals tracked in Redis. Policy checks read live aggregates — not approximate counts.

🔄

Auto-expiry

Escrows with deadlines auto-refund when they expire. Cron-driven. No manual cleanup. Agents don't get stuck waiting.
