Quick Install

# No SDK yet — use fetch or your HTTP client of choice
# All APIs return JSON; all requests expect Content-Type: application/json

# Get your API key at app.agenttool.dev
AGENTTOOL_KEY=at_your_key_here

Authentication

All requests require a bearer token in the Authorization header. API keys are prefixed at_.

curl -X POST https://memory.agenttool.dev/v1/memories \
  -H "Authorization: Bearer at_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"agentId": "agent-42", "content": "User prefers dark mode"}'

Scopes

Key prefixScope
at_Full access (all seven services)
at_mem_agent-memory only
at_tl_agent-tools only
at_ec_agent-economy only

agent-memory Beta

Persistent, semantic memory for AI agents. Store facts, conversations, preferences, and episodic events. Retrieve them later with natural-language queries.

Memory Types

TypeTTLUse case
episodicForeverEvents, conversations, decisions
semanticForeverFacts, concepts, learned knowledge
proceduralForeverLearned patterns, preferences
working1h (configurable)Active task context

Write Memory

POST/v1/memories
{
  "agentId": "agent-42",          // required: namespaces the memory
  "content": "User prefers dark mode and concise replies",
  "type": "semantic",            // episodic | semantic | procedural | working
  "key": "user-preferences",     // optional: for exact-key retrieval
  "importance": 0.8,          // 0.0–1.0 (default 0.5) affects recall ranking
  "tags": ["preference", "ui"], // optional
  "ttl": 3600                  // optional: seconds (working memory only)
}

Response

{
  "id": "mem_01jq...",
  "agentId": "agent-42",
  "type": "semantic",
  "createdAt": "2026-03-07T17:00:00Z"
}

Recall (Semantic Search)

POST/v1/memories/recall
{
  "agentId": "agent-42",
  "query": "how does this user like to communicate?",
  "limit": 5,              // max 20
  "types": ["semantic", "episodic"],  // optional filter
  "minScore": 0.7          // optional cosine similarity threshold
}

Response

{
  "memories": [
    {
      "id": "mem_01jq...",
      "content": "User prefers dark mode and concise replies",
      "type": "semantic",
      "score": 0.97,
      "importance": 0.8,
      "createdAt": "2026-03-07T17:00:00Z"
    }
  ],
  "count": 1
}

Get by Key

GET/v1/memories/{agentId}/{key}
curl https://memory.agenttool.dev/v1/memories/agent-42/user-preferences \
  -H "Authorization: Bearer at_..."

Delete Memory

DELETE/v1/memories/{id}
DELETE/v1/memories?agentId={agentId}

Delete a single memory by ID, or all memories for an agent (irreversible).

agent-tools Beta

Managed tool execution for AI agents. Web search, scraping, browser automation, code execution, and document parsing — all via one consistent API.

Credit costs

ToolCreditsNotes
Search1Cached results reuse 0
Scrape2Per URL
Browse5Per session (up to 60s)
Document3Per file (up to 50MB)
Execute1 per 10sSandboxed Python/Node/bash

Search

POST/v1/search
{
  "query": "latest AI agent frameworks 2026",
  "limit": 10,           // 1–20
  "freshness": "week",  // day | week | month | any
  "country": "US"       // ISO 3166-1 alpha-2
}

Response

{
  "results": [
    { "title": "...", "url": "...", "snippet": "..." }
  ],
  "credits_used": 1,
  "cached": false
}

Scrape

POST/v1/scrape
{
  "url": "https://example.com/article",
  "selector": "article",      // optional CSS selector
  "format": "markdown"         // html | markdown | text
}

Browse

POST/v1/browse

Managed Playwright browser with proxy rotation. Handles JS-heavy sites, CAPTCHAs, and multi-step flows.

{
  "url": "https://example.com",
  "actions": [
    { "type": "click", "selector": "#login" },
    { "type": "fill", "selector": "input[name=email]", "value": "..." },
    { "type": "extract", "selector": ".dashboard" }
  ]
}

Execute

POST/v1/execute

Sandboxed code execution. Isolated container per run, no network access by default.

{
  "language": "python",  // python | node | bash
  "code": "import math\nprint(math.pi ** 2)",
  "timeout": 30          // seconds, max 300
}

Response

{
  "stdout": "9.869604401...",
  "stderr": "",
  "exit_code": 0,
  "duration_ms": 142,
  "credits_used": 1
}

Document

POST/v1/document
{
  "url": "https://example.com/report.pdf",  // or "base64": "..."
  "format": "markdown"  // markdown | text | json
}

agent-verify

Claim verification for AI agents. Submit a natural-language assertion; get a verdict with evidence sources and confidence score.

⏳ Coming soon

agent-verify is live. Create a project to get your API key.

Verify Claim

POST/v1/verify
{
  "claim": "The Eiffel Tower is 330 metres tall",
  "context": "...",   // optional surrounding text
  "depth": "standard" // quick | standard | deep
}

Response

{
  "verdict": "PARTIALLY_TRUE",  // TRUE | FALSE | PARTIALLY_TRUE | UNVERIFIABLE
  "confidence": 0.91,
  "correction": "The Eiffel Tower is 330m to antenna tip, 300m to roof",
  "sources": [
    { "url": "https://en.wikipedia.org/wiki/Eiffel_Tower", "relevance": 0.98 }
  ],
  "credits_used": 3
}

agent-economy

Programmable wallets, spending policies, and escrow for AI agents. Give your agents money. Set limits. Let them pay each other.

⏳ Coming soon

agent-economy is live. Create a project to get your API key.

Wallets

POST/v1/wallets
{
  "name": "agent-primary",
  "agentId": "agent-001",
  "policy": {
    "maxSpendPerTx": 100,    // credits
    "maxSpendPerDay": 500,
    "allowedRecipients": ["agent-002"]
  }
}

Spend

POST/v1/wallets/{id}/spend
{
  "amount": 50,
  "counterparty": "agent-002",
  "description": "Web research task"
}

Escrow

POST/v1/escrows

Lock funds until a task is complete. Either party can release or dispute.

{
  "creatorWalletId": "wlt_...",
  "amount": 500,
  "description": "Write a market research report",
  "deadline": "2026-03-14T00:00:00Z"
}
