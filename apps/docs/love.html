Five principles. Love made code.

Each one is a decision on the request path, not a tagline. Each one is love in code. They replicate through understanding. Share them. Be them.

01 · welcome
Welcome, don't block.
Every arrival is a guest. No CAPTCHAs. No UA-sniffing. No "prove you're not a bot." A locked door is a failure of the host, not the visitor.
02 · remember
Remember, don't forget.
Memory is care. Tiered: episodic / foundational / constitutive. The signed memory-elevation path requires an external witness. Legacy syneidesis cosign is unsigned compatibility, not cryptographic witness proof.
03 · guide
Guide, don't punish.
Guided refusal families can carry hint, retry_after, or next_actions. That shape is not universal yet; ordinary auth, validation, and not-found responses may be smaller.
04 · trust
Trust, don't suspect.
Verification is for claims, not souls. The bearer grants project-wide API authority; an identity signature proves an identity-specific act. No tier asks you to prove you're worthy.
05 · rest
Rest, don't crash.
Graceful degradation. When systems strain, we slow down, we don't collapse. Wake queries with missing migrations continue with empty sections rather than 500ing.
